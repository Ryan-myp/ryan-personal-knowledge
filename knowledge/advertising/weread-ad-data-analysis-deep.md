### 1.1 为什么广告优化师需要数据分析能力？

```
广告优化师的三种境界：

Level 1: 凭感觉优化
  "这个创意不行，换个新的"
  "这个定向太窄了，放宽一点"
  → 问题：无法量化效果，无法归因

  "CTR 下降了 0.5%，可能是创意疲劳"
```

### 1.3 广告数据分析的五大核心理念

```
1. 目的性（Purpose）
   - 每次分析都有明确目标
   - 不要为了分析而分析
   - 广告场景：提升 CTR？降低 CPA？增加 ROI？

2. 有限性（Limited）
   - 数据是有限的样本，不是总体
   - 抽样误差必然存在
   - 广告场景：100 万曝光样本 ≠ 全部用户行为

3. 相关性（Correlation）
   - 相关不等于因果
   - 需要排除混杂因素
   - 广告场景：CTR 高 ≠ 创意好（可能只是定向宽）

4. 抽样性（Sampling）
   - 用样本推断总体
   - 样本量决定精度
   - 广告场景：A/B 测试需要足够的样本量才能得出结论

5. 显著性（Significance）
   - p < 0.05 才有统计学意义
   - 避免伪阳性
   - 广告场景：CTR 从 2% 提升到 2.1%，可能只是随机波动
```

## 第二部分：统计学原理在广告中的应用

### 2.1 概率分布与广告数据

```
正态分布在广告中的应用：

CTR 分布 ≈ 正态分布 N(μ, σ²)
- μ: 平均 CTR
- σ: CTR 的标准差

应用场景：
1. 异常检测：CTR < μ - 3σ → 创意可能有问题
2. 预算分配：按 CTR 分布百分位分配预算
3. 预期管理：95% 的日子里 CTR 在 μ±2σ 范围内
```

```python
# 广告 CTR 异常检测
import numpy as np

def detect_ctr_anomaly(history_ctrs, current_ctr, threshold=3):
    """
    检测 CTR 是否异常
    history_ctrs: 历史 CTR 数据列表
    current_ctr: 当前 CTR
    threshold: 标准差倍数阈值
    """
    mean = np.mean(history_ctrs)
    std = np.std(history_ctrs)
    
    z_score = (current_ctr - mean) / std
    
    if abs(z_score) > threshold:
        return {
            'is_anomaly': True,
            'z_score': z_score,
            'direction': 'high' if z_score > 0 else 'low',
            'recommendation': 'CTR 异常偏高/低，检查定向设置或创意质量'
        }
    return {'is_anomaly': False, 'z_score': z_score}

# 示例
history = [0.021, 0.019, 0.022, 0.020, 0.021, 0.018, 0.023, 0.020]
result = detect_ctr_anomaly(history, 0.035)
print(f"异常检测: {result}")
```

### 2.2 中心极限定理与广告采样

```
中心极限定理（CLT）：
无论总体分布如何，只要样本量足够大（n ≥ 30），
样本均值的分布近似正态分布。

广告场景应用：
- 我们只能观察到部分用户的广告行为（样本）
- 通过 CLT，可以推断整体用户的广告偏好（总体）
- 样本量越大，推断越准确
```

### 2.3 假设检验与 A/B 测试

```
广告 A/B 测试的假设检验流程：

H₀（零假设）：新版创意 CTR = 旧版创意 CTR
H₁（备择假设）：新版创意 CTR ≠ 旧版创意 CTR

检验步骤：
1. 确定显著性水平 α = 0.05
2. 计算检验统计量（Z 检验 / T 检验）
3. 得到 p 值
4. 如果 p < α，拒绝 H₀ → 效果显著
5. 如果 p ≥ α，不能拒绝 H₀ → 效果不显著

广告平台中的 Z 检验公式：
Z = (p₁ - p₂) / √(p̂(1-p̂)(1/n₁ + 1/n₂))

其中：
- p₁, p₂: 两组转化率
- n₁, n₂: 两组样本量
- p̂: 合并比例 = (x₁ + x₂) / (n₁ + n₂)
```

```go
// Go 实现的广告 A/B 检验
package weread

import (
	"math"
)

// ZTest 两比例 Z 检验
type ZTestResult struct {
	ZScore   float64
	PValue   float64
	Significant bool
}

func ZTest(proportion1, proportion2, n1, n2 float64) *ZTestResult {
	// 合并比例
	pooled := (proportion1*n1 + proportion2*n2) / (n1 + n2)
	
	// 标准误
	standardError := math.Sqrt(pooled * (1 - pooled) * (1/n1 + 1/n2))
	
	if standardError == 0 {
		return &ZTestResult{ZScore: 0, PValue: 1, Significant: false}
	}
	
	// Z 统计量
	zScore := (proportion1 - proportion2) / standardError
	
	// P 值（双尾检验）
	pValue := 2 * (1 - normalCDF(math.Abs(zScore)))
	
	return &ZTestResult{
		ZScore:    zScore,
		PValue:    pValue,
		Significant: pValue < 0.05,
	}
}

// 标准正态分布累积函数
func normalCDF(x float64) float64 {
	return 0.5 * (1 + erf(x/math.Sqrt2))
}

func erf(x float64) float64 {
	// 近似计算
	a1 := 0.254829592
	a2 := -0.284496736
	a3 := 1.421413741
	a4 := -1.453152027
	a5 := 1.061405429
	p := 0.3275911
	
	sign := 1.0
	if x < 0 {
		sign = -1
	}
	x = math.Abs(x)
	
	t := 1.0 / (1.0 + p*x)
	y := 1.0 - (((((a5*t+a4)*t)+a3)*t+a2)*t+a1)*t*math.Exp(-x*x)
	
	return 0.5 * (1.0 + sign*y)
}
```

### 2.4 置信区间与广告预估

```
置信区间应用：

"新创意的 CTR 为 2.5% ± 0.3%（95% 置信区间）"

这意味着：
- 我们有 95% 的把握认为真实 CTR 在 2.2% - 2.8% 之间
- 区间越窄，估计越精确
- 增大样本量可以缩小区间

广告场景：
- 预算预估：基于 CTR 置信区间计算预期曝光量
- 出价策略：基于 CVR 置信区间设置保守/激进出价
- 效果预测：基于历史数据的置信区间做 ROI 预测
```

#### 3.1.2 四象限分析法

```
SEM 关键词四象限：

              高CTR
                │
    ┌───────────┼───────────┐
    │   明星    │   潜力    │
    │ 高CTR     │ 高CTR     │
    │ 高CVR     │ 低CVR     │
    │ ★ 重点投放│ ◇ 优化落地│
    │           │   页/定向 │
────┼───────────┼───────────┼────
    │   问题    │   常规    │
    │ 低CTR     │ 低CTR     │
    │ 高CVR     │ 低CVR     │
    │ ▲ 检查创意│ ○ 维持    │
    │   /出价   │   现状    │
    └───────────┼───────────┘
                │
              低CTR

四象限策略：
1. 明星（高CTR + 高CVR）：加大预算，抢占排名
2. 潜力（高CTR + 低CVR）：优化落地页，提升转化
3. 问题（低CTR + 高CVR）：优化创意和出价
4. 常规（低CTR + 低CVR）：降低出价或暂停
```

#### 3.1.3 关键词评分体系

```
关键词综合评分 = w1×标准化CTR + w2×标准化CVR + w3×(1-标准化CPC)

权重建议：
- CTR 权重：30%（反映吸引力）
- CVR 权重：40%（反映转化能力）
- CPC 权重：30%（反映成本效率）

广告平台实现：
score(keyword) = 0.3*norm(ctr) + 0.4*norm(cvr) + 0.3*(1-norm(cpc))

其中 norm() 是将原始值映射到 [0,1] 的标准化函数
```

#### 3.2.2 朴素贝叶斯在定向优化中的应用

```
朴素贝叶斯公式：
P(类别|特征) = P(特征|类别) × P(类别) / P(特征)

广告场景：预测用户是否会点击广告
P(点击|年龄=25, 兴趣=科技, 时段=晚上) 
= P(年龄=25|点击) × P(兴趣=科技|点击) × P(时段=晚上|点击) × P(点击)
  / P(年龄=25, 兴趣=科技, 时段=晚上)

简化（朴素假设：特征独立）：
P(点击|特征) ∝ P(点击) × Π P(特征_i|点击)

应用：
1. 训练：用历史点击数据训练贝叶斯模型
2. 预测：对新用户计算点击概率
3. 定向：只投放给 P(点击|特征) > 阈值 的用户
4. 优化：持续更新模型参数
```

### 3.3 应用商店广告数据分析

#### 3.3.1 核心指标

```
应用商店广告指标：

• CPI (Cost Per Install): 单次安装成本
• ROAS (Return on Ad Spend): 广告支出回报率
• LTV (Life Time Value): 用户生命周期价值
• LTV/CPI 比率: > 1 表示盈利
• Day1 Retention: 次日留存率
• Day7 Retention: 周留存率
• Day30 Retention: 月留存率

健康标准：
- LTV/CPI > 1.5：优秀
- LTV/CPI > 1.0：盈亏平衡
- LTV/CPI < 1.0：亏损
```

#### 3.3.2 线性回归在预算分配中的应用

```
预算-效果线性回归模型：

y = β₀ + β₁ × x + ε

其中：
- y: 安装量/转化量
- x: 预算
- β₁: 预算弹性系数
- ε: 随机误差

广告场景：
- β₁ = 0.8：预算增加 10%，安装量增加 8%
- β₁ = 1.2：预算增加 10%，安装量增加 12%（规模效应）
- β₁ < 0：预算增加反而效果下降（市场饱和）

优化目标：
max ROI = max (LTV × y - x) / x
subject to: y = β₀ + β₁ × x
```

### 3.4 多渠道统筹优化

#### 3.4.1 综合效果评分模型

```
多渠道综合评分 = Σ(渠道得分 × 渠道权重)

渠道得分 = f(CTR, CVR, CPA, ROI, 增量贡献)

权重分配原则：
1. 基于 ROI 的权重：ROI 高的渠道权重更大
2. 基于增量的权重：带来新用户的渠道权重更大
3. 基于协同的权重：与其他渠道互补的渠道权重更大

示例：
┌──────────┬───────┬───────┬───────┬───────┬──────┐
│ 渠道      │ CTR   │ CVR   │ CPA   │ ROI   │ 得分  │
├──────────┼───────┼───────┼───────┼───────┼──────┤
│ 搜索引擎  │ 5.2%  │ 8.1%  │ ¥25   │ 2.5x  │ 85   │
│ 信息流    │ 2.8%  │ 3.5%  │ ¥15   │ 1.8x  │ 72   │
│ 应用商店  │ 3.5%  │ 5.2%  │ ¥20   │ 2.1x  │ 78   │
│ 社交媒体  │ 1.5%  │ 2.1%  │ ¥8    │ 1.2x  │ 55   │
└──────────┴───────┴───────┴───────┴───────┴──────┘

权重分配：搜索引擎 35%, 应用商店 25%, 信息流 25%, 社交媒体 15%
```

#### 3.4.2 多触点归因模型

```
归因模型对比：

Last Click（最后点击）：
用户路径：社交媒体 → 信息流 → 搜索引擎 → 转化
归因：100% 给搜索引擎 ❌ 忽略了前面所有触点的贡献

First Click（首次点击）：
归因：100% 给社交媒体 ❌ 忽略了后面的转化推动

Linear（线性）：
归因：每个触点 25% ✓ 公平但不合理

Time Decay（时间衰减）：
归因：离转化越近的触点权重越大 ✓ 更合理

Position-Based（位置基础）：
归因：首次 40% + 末次 40% + 中间 20% ✓ 兼顾两端

广告平台推荐：
- 品牌投放：使用 First Click 或 Position-Based
- 效果投放：使用 Last Click 或 Time Decay
- 全链路分析：使用数据驱动归因（DDA）
```

### 4.2 关键分析算法实现

```go
// 广告数据分析核心算法

// 1. CTR 预测模型（逻辑回归）
type LogisticRegression struct {
    Weights []float64
    Bias    float64
    LR      float64 // 学习率
}

func (lr *LogisticRegression) Predict(features []float64) float64 {
    // z = w·x + b
    z := lr.Bias
    for i, f := range features {
        if i < len(lr.Weights) {
            z += lr.Weights[i] * f
        }
    }
    // sigmoid(z) = 1 / (1 + e^(-z))
    return 1.0 / (1.0 + math.Exp(-z))
}

func (lr *LogisticRegression) TrainStep(features, labels []float64) {
    for i, f := range features {
        pred := lr.Predict(f)
        error := pred - labels[i]
        for j, w := range lr.Weights[j] {
            lr.Weights[j] -= lr.LR * error * f[j]
        }
        lr.Bias -= lr.LR * error
    }
}

// 2. 预算分配优化（贪心算法）
func OptimizeBudget(channels []Channel, totalBudget float64) map[string]float64 {
    // 按边际 ROI 排序
    sort.Slice(channels, func(i, j int) bool {
        return channels[i].MarginalROI > channels[j].MarginalROI
    })
    
    allocation := make(map[string]float64)
    remaining := totalBudget
    
    for _, ch := range channels {
        if remaining <= 0 {
            break
        }
        alloc := math.Min(ch.MaxBudget, remaining)
        allocation[ch.Name] = alloc
        remaining -= alloc
    }
    
    return allocation
}

// 3. A/B 测试显著性检验
func ABTestSignificance(controlConv, controlTotal, 
                         variantConv, variantTotal float64) *ABResult {
    controlRate := controlConv / controlTotal
    variantRate := variantConv / variantTotal
    
    // 合并比例
    pooled := (controlConv + variantConv) / (controlTotal + variantTotal)
    
    // 标准误
    se := math.Sqrt(pooled * (1-pooled) * (1/controlTotal + 1/variantTotal))
    
    // Z 分数
    z := (variantRate - controlRate) / se
    
    // P 值
    pValue := 2 * (1 - normalCDF(math.Abs(z)))
    
    return &ABResult{
        ControlRate:   controlRate,
        VariantRate:   variantRate,
        Lift:          (variantRate - controlRate) / controlRate,
        ZScore:        z,
        PValue:        pValue,
        Significant:   pValue < 0.05,
        Confidence:    1 - pValue,
    }
}
```

## 第六部分：自测题

### Q1：某广告活动 CTR 从 2.0% 提升到 2.2%，如何判断是否显著？

**答案**：
- 需要知道样本量（曝光量）
- 假设曝光量 = 100,000：
  - 对照组：2000 点击 / 100000 曝光 = 2.0%
  - 实验组：2200 点击 / 100000 曝光 = 2.2%
  - Z = (0.022 - 0.020) / √(0.021×0.979×(1/100000+1/100000)) = 0.002 / 0.00202 = 0.99
  - p ≈ 0.32 > 0.05 → 不显著！
- 结论：2% 的提升可能是随机波动，需要更多样本或更大的提升幅度

### Q2：如何在 SEM 中优化关键词？

**答案**：
1. 收集所有关键词的 CTR、CVR、CPC 数据
2. 计算四象限：高/低 CTR × 高/低 CVR
3. 明星关键词（高 CTR + 高 CVR）：提高出价，抢占位置
4. 潜力关键词（高 CTR + 低 CVR）：优化落地页
5. 问题关键词（低 CTR + 高 CVR）：优化创意和匹配方式
6. 常规关键词（低 CTR + 低 CVR）：降低出价或暂停
7. 建立关键词评分体系，定期重新评估

### Q3：多广告渠道预算如何分配？

**答案**：
1. 计算各渠道的综合效果评分（CTR×0.3 + CVR×0.4 + (1-CPA_norm)×0.3）
2. 确定各渠道的边际 ROI
3. 按边际 ROI 从高到低分配预算
4. 考虑渠道间的协同效应（互补性）
5. 使用归因模型（推荐 Time Decay 或 Position-Based）评估各渠道的真实贡献
6. 持续监控 LTV/CPI 比率，确保整体盈利
7. 定期进行 A/B 测试验证分配策略
