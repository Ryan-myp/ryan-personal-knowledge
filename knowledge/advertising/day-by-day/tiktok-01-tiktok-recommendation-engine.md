## 第一部分：TikTok 推荐系统架构

### 1.1 TikTok 的双塔模型

```
TikTok 的核心竞争力是它的推荐系统。
广告推荐与内容推荐共享同一个底层架构。

双塔模型 (Dual-Encoder) 架构:

┌─────────────────────────────────────────────────────────────────┐
│                  Dual-Encoder Architecture                       │
│                                                                  │
│  ┌─────────────────────┐    ┌─────────────────────┐            │
│  │    User Tower       │    │    Item Tower       │            │
│  │                     │    │                     │            │
│  │  User Features:     │    │  Item Features:     │            │
│  │  ├── 历史行为        │    │  ├── 视频特征        │            │
│  │  ├── 兴趣标签        │    │  ├── 创作者特征      │            │
│  │  ├── 设备信息        │    │  ├── 音乐/标签       │            │
│  │  └── 人口统计        │    │  └── 互动历史        │            │
│  │                     │    │                     │            │
│  │  Embedding → e_u    │    │  Embedding → e_v    │            │
│  │  (128-dim)          │    │  (128-dim)          │            │
│  └─────────────────────┘    └─────────────────────┘            │
│             │                         │                          │
│             └──────→ Cosine Similarity ←──────────┘             │
│                         Score ∈ [-1, 1]                         │
│                                                                  │
│  Score(u, v) = e_u · e_v / (||e_u|| × ||e_v||)                  │
│                                                                  │
│  排序: 按 Score 降序 → Top N 个视频/广告展示给用户               │
└─────────────────────────────────────────────────────────────────┘

双塔模型的优势:
├─ 离线训练，在线推理快（128 维向量点积）
├─ 可扩展到数十亿级别
└─ 用户和物品独立编码，可缓存
```

### 1.2 TikTok 推荐系统的完整流程

```
TikTok 推荐系统 Pipeline:

┌─────────────────────────────────────────────────────────────────┐
│                   TikTok 推荐 Pipeline                           │
│                                                                  │
│  Step 1: Candidate Generation (召回)                              │
│  ──────────────────────────────────────────────────────────────   │
│  从数亿视频中召回 ~500 个给用户                                   │
│                                                                  │
│  召回策略:                                                       │
│  ├── 协同过滤 (CF): "喜欢这个的人也看了..."                       │
│  ├── 基于内容 (Content-based): 相似标签/创作者                     │
│  ├── 热门召回: 全局热门视频                                       │
│  ├── 关注召回: 关注的创作者的新视频                                 │
│  ├── 搜索召回: 搜索过的相关视频                                    │
│  └── 实时召回: 用户当前会话的行为                                  │
│                                                                  │
│  Step 2: Pre-Ranking (预排序)                                    │
│  ──────────────────────────────────────────────────────────────   │
│  从 ~500 个中选出 ~50 个                                         │
│                                                                  │
│  模型: 轻量级 DNN (~10M 参数)                                    │
│  输出: 初步的 Engagement Score                                   │
│  耗时: ~5ms                                                      │
│                                                                  │
│  Step 3: Ranking (精排序)                                        │
│  ──────────────────────────────────────────────────────────────   │
│  从 ~50 个中选出最终 Top 10 (实际展示 1 个)                       │
│                                                                  │
│  模型: 复杂 DNN (~100M 参数)                                     │
│  输出: 精确的 pWatch, pLike, pComment, pShare, pFollow           │
│  耗时: ~20ms                                                     │
│                                                                  │
│  Step 4: Re-Ranking (重排)                                       │
│  ──────────────────────────────────────────────────────────────   │
│  最终过滤和排序                                                  │
│                                                                  │
│  策略:                                                           │
│  ├── 多样性控制: 不同创作者、不同类别                              │
│  ├── 频率控制: 同一创作者不超过 N 个                              │
│  ├── 新鲜度: 优先展示新视频                                       │
│  └── 广告插入: 在有机视频中插入广告                                │
│                                                                  │
│  Total Pipeline Latency: ~50ms (用户无感知)                       │
└─────────────────────────────────────────────────────────────────┘
```

### 1.3 召回阶段的深度分析

```
召回阶段 (Candidate Generation) 是推荐系统的瓶颈：

问题: 从数十亿视频中召回 ~500 个

解决方案: 多级召回

Level 1: 精确召回 (Exact Match)
├─ 基于用户 ID 召回: 关注列表、历史互动
├─ 基于内容 ID 召回: 相似视频（基于标签/类别）
└─ 耗时: ~1ms

Level 2: 向量召回 (Vector Search)
├─ 用户向量 e_u 与所有视频向量 e_v 做近似最近邻搜索
├─ 使用 ANN 索引 (如 HNSW)
├─ 召回 ~200 个
└─ 耗时: ~5ms

Level 3: 规则召回 (Rule-based)
├─ 热门视频 (全局热度)
├─ 新视频 (新鲜度)
├─ 关注的新视频
└─ 耗时: ~2ms

Level 4: 实时召回 (Real-time)
├─ 基于用户当前会话的实时行为
├─ 当前视频的相似内容
└─ 耗时: ~2ms

总召回: ~500 个
总耗时: ~10ms
```

### 2.1 广告与有机内容的混排

```
TikTok 广告如何与有机内容混排？

┌───────────────────────────────────────────────────────────────┐
│              TikTok 广告插入机制                               │
│                                                               │
│  方案 A: 广告作为独立候选 (Recommended)                        │
│  ────────────────────────────────────────────────────────────  │
│  广告和有机内容走同一个推荐 Pipeline                           │
│                                                               │
│  1. 广告也经过召回 → 预排序 → 精排序                          │
│  2. 广告与有机内容在同一池子中竞争 Score                       │
│  3. 最终根据 Score 混合展示                                    │
│                                                               │
│  优点:                                                         │
│  ├── 广告获得精准的受众匹配                                    │
│  ├── 用户体验一致（都是"刷"到的）                              │
│  └── 广告效果最好                                              │
│                                                               │
│  缺点:                                                         │
│  ├── 广告需要高质量的素材才能竞争                               │
│  └── 广告主的出价/ bid 影响展示                                │
│                                                               │
│  方案 B: 固定比例插入 (Legacy)                                 │
│  ────────────────────────────────────────────────────────────  │
│  每 N 个有机视频插入 1 个广告                                  │
│                                                               │
│  优点: 广告主保证曝光                                          │
│  缺点: 用户体验差，广告相关性低                                 │
│                                                               │
│  现状: TikTok 主要采用方案 A（广告作为候选）                    │
└───────────────────────────────────────────────────────────────┘
```

### 2.2 Spark Ads：有机内容变广告

```
Spark Ads 是 TikTok 的特色广告形式：

Spark Ads 的本质:
├─ 使用已有的有机视频作为广告素材
├─ 视频本身已经经过推荐系统验证（高互动）
└─ 广告主为高互动视频"投流"

Spark Ads 的优势:
├─ 真实性: 不是专门制作的广告，用户更容易信任
├─ 效果验证: 有机视频已经有互动数据证明质量
└─ 成本低: 不需要重新制作视频素材

Spark Ads 的实现:
1. 有机视频发布后，TikTok 推荐系统自动评估其表现
2. 高表现的视频进入"广告候选池"
3. 广告主可以选择哪些视频进行 Spark Ads 推广
4. 推广时，视频仍然保持有机内容的外观

Spark Ads 的排名机制:
├─ Spark Ads 使用有机视频的 Score 作为基础
├─ 加上广告出价: Final_Score = Organic_Score × Bid_Multiplier
├─ 有机分数高 + 出价合理 = 最佳效果
└─ 有机分数低 = 即使出价高也难以获得好效果
```

### 2.3 广告与内容的协同过滤

```
TikTok 广告如何利用协同过滤提高效果？

协同过滤在广告中的应用:

用户-物品协同过滤 (User-Item CF):
├─ 如果用户 A 喜欢视频 X，喜欢视频 Y
├─ 用户 B 也喜欢视频 X
└─ 预测用户 B 也喜欢视频 Y

广告场景:
├─ 用户 A 点击了广告 X → 转化
├─ 用户 B 的行为模式与用户 A 相似
├─ 广告 X 应该也推荐给 B

实现:
├─ 用户特征向量 e_u 编码了用户的行为模式
├─ 物品特征向量 e_v 编码了视频/广告的特征
├─ 点积 e_u · e_v 衡量用户-物品的匹配度
└─ 高匹配度的广告/视频被推荐

广告协同过滤的特殊性:
├─ 冷启动: 新广告没有历史数据
├─ 解决方案: 使用内容特征（视频分类、标签、创作者）
└─ 数据稀疏: 广告转化数据远少于有机互动
```

## 第三部分：TikTok 竞价与出价

### 3.1 TikTok 的竞价机制

```
TikTok 广告竞价机制：

OCPM (Optimized CPM) 机制:

┌───────────────────────────────────────────────────────────────┐
│                   OCPM Bidding Model                          │
│                                                               │
│  用户选择优化目标:                                             │
│  ├── Conversion (转化)                                        │
│  ├── Link Click (点击)                                        │
│  └── Impression (展示)                                        │
│                                                               │
│  系统自动优化:                                                 │
│  ┌─────────────────────────────────────────────────────────┐  │
│  │  Bid_Score = Bid × pAction × Bid_Adjustment             │  │
│  │                                                       │  │
│  │  其中:                                                 │  │
│  │  ├── Bid: 用户设置的 CPM 出价                           │  │
│  │  ├── pAction: 预测的行动概率（转化/点击/展示）           │  │
│  │  └── Bid_Adjustment: 基于竞争环境的调整                  │  │
│  └─────────────────────────────────────────────────────────┘  │
│                                                               │
│  排序: 按 Bid_Score 降序                                       │
│  结算: 第二价格 (Second Price)                                 │
└───────────────────────────────────────────────────────────────┘

OCPM 的核心优势:
├─ 用户只需设置 CPM 出价
├─ 系统自动根据 pAction 调整实际 CPC
├─ 高 pAction 的广告获得更好的展示
└─ 低 pAction 的广告实际 CPC 更低
```

### 3.2 pCVR 预测模型

```
TikTok 的 pCVR 预测模型：

pCVR = P(user converts | user, ad, context)

模型输入:
├─ 用户特征:
│   ├── 历史转化行为
│   ├── 兴趣标签
│   ├── 设备类型
│   └── 地理位置
├─ 广告特征:
│   ├── 视频分类（产品类/娱乐类/教育类）
│   ├── 视频标签
│   ├── 创作者类型
│   └── 历史 CTR/CVR
└─ 上下文特征:
    ├── 时间
    ├── 网络类型
    └── 设备

模型架构:
├─ Embedding Layer: 离散特征 → 稠密向量
├─ Deep Network: 多层 DNN 提取特征
├─ Attention: 关注关键特征
└─ Output: Sigmoid → [0, 1]

训练数据:
├─ Positive: 用户点击广告并转化
├─ Negative: 用户点击广告但未转化
└─ Hard Negative: 用户展示广告但未点击
```

### 3.3 出价优化策略

```
TikTok 出价优化策略：

策略 1: Cost Cap (成本上限)
├─ 设置目标 CPA
├─ 系统自动调整出价以接近目标
├─ 允许短期波动（±15%）
└─ 长期稳定在目标 CPA

策略 2: Bid Cap (出价上限)
├─ 设置最高 CPC/CPM
├─ 系统不会超过此出价
├─ 适合预算有限的广告主
└─ 可能导致展示不足

策略 3: Lowest Cost (最低成本)
├─ 系统自动出价以获取最多转化
├─ 不设上限
├─ 适合预算充足、追求规模
└─ CPA 可能波动

策略 4: Target ROAS (目标 ROAS)
├─ 设置目标回报率
├─ 系统优先展示高价值转化
├─ 适合电商
└─ 需要充分的转化数据
```

## 第四部分：TikTok 广告优化实战

### 4.1 视频素材优化

```
TikTok 广告素材优化原则：

1. 前 3 秒 (Hook)
   ─────────────────────────────
   ── 最重要的 3 秒!
   ── 必须抓住注意力
   ── 方法: 悬念、问题、视觉冲击、名人

2. 原生感 (Authenticity)
   ─────────────────────────────
   ── UGC 风格 > 专业制作
   ── 真实用户 > 演员
   ── 竖屏 > 横屏
   ── 自然 > 过度包装

3. 音乐 (Music)
   ─────────────────────────────
   ── 使用 TikTok Trending 音乐
   ── 音乐与内容匹配
   ── 音乐驱动情绪

4. CTA (Call to Action)
   ─────────────────────────────
   ── 明确告诉用户做什么
   ── "Shop Now", "Learn More"
   ── 放在视频末尾

5. 长度
   ─────────────────────────────
   ── 15-30 秒最佳
   ── 超过 60 秒转化率下降
   ── 短视频 > 长视频（通常）
```

### 4.2 受众定向策略

```
TikTok 受众定向策略：

1. 第一方受众 (Best)
   ─────────────────────────────
   ├── 上传 CRM 数据
   ├── 网站像素数据
   └── 应用用户数据

2. Lookalike Audience
   ─────────────────────────────
   ├── 基于高价值用户创建
   ├── TikTok 自动找到相似用户
   └── 质量取决于种子数据

3. Interest-based
   ─────────────────────────────
   ├── 基于用户兴趣标签
   ├── 选择相关类别
   └── 效果中等

4. Broad (Advantage+)
   ─────────────────────────────
   ├── 不设定向限制
   ├── TikTok 自动寻找用户
   └── 效果取决于创意质量
```

### 4.1 视频表现分析

```python
# TikTok 视频表现分析
def analyze_video_performance(impressions, plays_3s, full_plays,
                                likes, comments, shares, clicks, conversions):
    """分析视频表现指标"""
    
    play_3s_rate = plays_3s / max(impressions, 1)
    completion_rate = full_plays / max(plays_3s, 1)
    like_rate = likes / max(impressions, 1)
    share_rate = shares / max(impressions, 1)
    ctr = clicks / max(impressions, 1)
    cvr = conversions / max(clicks, 1)
    
    return {
        'play_3s_rate': play_3s_rate,
        'completion_rate': completion_rate,
        'like_rate': like_rate,
        'share_rate': share_rate,
        'ctr': ctr,
        'cvr': cvr,
    }

# 使用示例
result = analyze_video_performance(
    impressions=100000,
    plays_3s=35000,
    full_plays=15000,
    likes=5000,
    comments=500,
    shares=1000,
    clicks=3000,
    conversions=150,
)
for k, v in result.items():
    print(f"{k}: {v:.4f}")
```
