# TCP 拥塞控制深度：从 Reno 到 BBR 的演进

> 从经典算法到现代拥塞控制，逐层解析 TCP 拥塞控制原理

---

## 第一部分：TCP 拥塞控制基础

### 拥塞控制的核心思想

```
拥塞控制目标：
┌─────────────────────────────────────────────────────────────────────┐
│ 1. 避免网络过载                                                      │
│    ├── 路由器缓冲区溢出导致丢包                                     │
│    ├── 丢包触发重传增加网络负担                                     │
│    └── 恶性循环导致网络崩溃                                         │
│                                                                     │
│ 2. 公平共享                                                          │
│    ├── 多流共享链路带宽                                             │
│    ├── 避免单一流独占带宽                                           │
│    └── 保证用户体验                                                 │
│                                                                     │
│ 3. 高效利用                                                          │
│    ├── 最大化链路利用率                                             │
│    ├── 最小化延迟                                                   │
│    └── 平衡吞吐量和延迟                                             │
└─────────────────────────────────────────────────────────────────────┘
```

### 拥塞窗口（CWND）

```
拥塞窗口管理：
┌─────────────────────────────────────────────────────────────────────┐
│ CWND：决定发送方可以发送的最大数据量                                 │
│                                                                     │
│ 状态转换：                                                          │
│ • 慢启动：CWND 指数增长                                            │
│ • 拥塞避免：CWND 线性增长                                          │
│ • 快速重传：收到 3 个重复 ACK                                      │
│ • 快速恢复：半开状态                                               │
│ • 超时：回到慢启动                                                 │
└─────────────────────────────────────────────────────────────────────┘
```

## 第二部分：经典拥塞控制算法

### Tahoe/Reno/Vegas 对比

```
经典算法对比：
┌────────────────┬────────────┬────────────┬────────────┐
│     特性       │  Tahoe     │  Reno      │  Vegas     │
├────────────────┼────────────┼────────────┼────────────┤
│ 慢启动         │ 指数增长   │ 指数增长   │ 指数增长   │
│ 拥塞避免       │ 线性增长   │ 线性增长   │ 延迟驱动   │
│ 快速重传       │ 不支持     │ 支持       │ 支持       │
│ 快速恢复       │ 不支持     │ 支持       │ 支持       │
│ 丢包检测       │ 超时       │ 3x DupACK  │ RTT 变化   │
│ 适用场景       │ 早期网络   │ 现代网络   │ 低延迟网络 │
└────────────────┴────────────┴────────────┴────────────┘
```

### Reno 算法详解

```
Reno 拥塞控制：
┌─────────────────────────────────────────────────────────────────────┐
│ 1. 慢启动（Slow Start）                                              │
│    ├── 初始 CWND = 1 MSS                                            │
│    ├── 每 RTT CWND 翻倍                                             │
│    └── 达到 ssthresh 进入拥塞避免                                   │
│                                                                     │
│ 2. 拥塞避免（Congestion Avoidance）                                  │
│    ├── 每 RTT CWND += 1/CWND                                        │
│    ├── 近似线性增长                                                 │
│    └── 达到阈值触发拥塞处理                                         │
│                                                                     │
│ 3. 快速重传（Fast Retransmit）                                       │
│    ├── 收到 3 个重复 ACK                                            │
│    ├── 立即重传丢失报文                                             │
│    └── ssthresh = CWND / 2                                         │
│                                                                     │
│ 4. 快速恢复（Fast Recovery）                                         │
│    ├── 进入快速恢复状态                                             │
│    ├── CWND = ssthresh + 3 MSS                                     │
│    └── 每收到一个重复 ACK，CWND += 1 MSS                            │
└─────────────────────────────────────────────────────────────────────┘
```

## 第三部分：现代拥塞控制算法

### CUBIC 算法

```
CUBIC 拥塞控制：
┌─────────────────────────────────────────────────────────────────────┐
│ 设计目标：                                                          │
│ • 高带宽延迟积（BDP）网络优化                                       │
│ • 快速收敛到公平点                                                   │
│ • 减少振荡                                                         │
│                                                                     │
│ 核心机制：                                                          │
│ 1. CUBIC 函数                                                        │
│    ├── W_cubic(t) = C*(t-T)^3 + W_max                              │
│    ├── C: 收敛速度常数                                              │
│    ├── T: 到达 W_max 的时间                                        │
│    └── W_max: 上次拥塞时的窗口大小                                  │
│                                                                     │
│ 2. 三次曲线特点                                                      │
│    ├── 收敛速度快                                                   │
│    ├── 稳定性好                                                     │
│    └── 公平性佳                                                     │
│                                                                     │
│ 3. 与 Reno 的比较                                                    │
│    ├── Reno：线性增长，适合低带宽                                   │
│    └── CUBIC：三次增长，适合高带宽                                  │
└─────────────────────────────────────────────────────────────────────┘
```

### BBR 算法

```
BBR 拥塞控制：
┌─────────────────────────────────────────────────────────────────────┐
│ 核心思想：                                                          │
│ • 不再依赖丢包作为拥塞信号                                         │
│ • 直接测量网络路径的带宽和延迟                                     │
│ • 主动维持最佳 CWND                                                │
│                                                                     │
│ 四个阶段：                                                          │
│ 1. Startup（启动）                                                  │
│    ├── 快速填充网络                                                 │
│    ├── CWND 快速增长                                                │
│    └── 目标：找到 BDP                                              │
│                                                                     │
│ 2. Drain（排空）                                                    │
│    ├── 减少队列积压                                                 │
│    ├── 降低 RTT                                                     │
│    └── 进入 steady state                                          │
│                                                                     │
│ 3. Probe_BW（带宽探测）                                             │
│    ├── 周期性脉冲发送                                               │
│    ├── 测量最大带宽                                                 │
│    └── 维持高利用率                                                │
│                                                                     │
│ 4. Probe_RTT（RTT 探测）                                            │
│    ├── 短暂停止发送                                                 │
│    ├── 测量最小 RTT                                                 │
│    └── 重置 CWND                                                  │
│                                                                     │
│ 优势：                                                              │
│ • 低延迟：减少排队延迟                                             │
│ • 高吞吐：充分利用带宽                                             │
│ • 公平性：与其他流共享带宽                                         │
│ • 稳定性：避免振荡                                                 │
└─────────────────────────────────────────────────────────────────────┘
```

## 第四部分：拥塞控制实践

### 拥塞控制选择

```
算法选择指南：
┌─────────────────────────────────────────────────────────────────────┐
│ 场景：                                                              │
│ • 传统 TCP 应用：CUBIC（Linux 默认）                                │
│ • 低延迟要求：BBR                                                   │
│ • 高带宽网络：BBR                                                   │
│ • 移动网络：BBR（抗抖动能力强）                                     │
│ • CDN 分发：CUBIC                                                   │
│ • 实时应用：BBR（减少排队延迟）                                     │
│                                                                     │
│ 配置建议：                                                          │
│ • Linux：net.ipv4.tcp_congestion_control = bbr                    │
│ • Windows：默认 BBRv2                                               │
│ • macOS：默认 CUBIC                                                 │
└─────────────────────────────────────────────────────────────────────┘
```

## 第五部分：自测题

### Q1: TCP 拥塞控制的三个主要状态？

**A**: 慢启动、拥塞避免、快速恢复。

### Q2: BBR 相比传统算法的优势？

**A**: 不依赖丢包、直接测量带宽和延迟、低延迟、高吞吐、适合移动网络。

### Q3: CUBIC 的核心特点？

**A**: 三次曲线增长、适合高带宽延迟积网络、收敛快、稳定性好。
