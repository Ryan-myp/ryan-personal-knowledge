## 二、架构模块学习路径

### 第 1 阶段：架构基础（2-3 周）

**目标**: 掌握架构设计方法论和基本思维

**学习顺序**:
1. 《从程序员到架构师》 → 核心：实战经验总结
2. 《架构师应该知道的37件事》 → 核心：知识体系全景
3. 《搞定系统设计》 → 核心：系统设计面试

**检验重点**:
- 能解释架构设计的 5 个核心维度（性能、可用性、可扩展性、安全性、可维护性）
- 能画出常见系统的架构图
- 能解释 CAP 定理的实际应用

### 第 2 阶段：架构模式（3-4 周）

**目标**: 掌握常用架构模式和设计模式

**学习顺序**:
1. 《设计模式》 → 23 种设计模式
2. 《企业应用架构模式》 → 架构模式
3. 《微服务设计》 → 微服务架构
4. 《架构解密：从分布式到微服务》 → 架构演进

**检验重点**:
- 能识别常见架构模式（分层、MVC、CQRS、EventSourcing）
- 能解释设计模式在 Go 中的实现
- 能画出微服务架构图

### 第 3 阶段：分布式系统（4-6 周）

**目标**: 掌握分布式系统原理和架构

**学习顺序**:
1. 《分布式系统架构》 → 架构策略
2. 《分布式架构原理与实践》 → 工程实践
3. 《从Paxos到Zookeeper》 → 一致性协议
4. 《凤凰架构》 → 大型分布式系统设计

**检验重点**:
- 能解释 CAP、BASE 理论
- 能实现分布式事务（2PC/Saga/TCC）
- 能设计高可用架构

### 第 4 阶段：专业深化（2-3 周）

**目标**: 掌握特定领域的架构技术

**学习顺序**:
1. 《Kubernetes快速入门》 → K8s 容器编排
2. 《Redis设计与实现》 → Redis 源码
3. 《凤凰架构》 → 大型分布式系统设计

**检验重点**:
- 能设计基于 K8s 的部署架构
- 能解释 Redis 集群原理
- 能画出复杂系统的架构图

## 四、学习监督机制

### 每周检查

1. **阅读进度**: 本周读了哪本书？读了多少页？
2. **知识沉淀**: 是否写了读书笔记？
3. **实践应用**: 能否用学到的知识解决实际问题？

### 月度评估

1. **模块完成度**: 当前阶段的学习进度
2. **知识掌握度**: 通过检验题目的得分
3. **实践能力**: 是否能画出系统架构图

### 季度回顾

1. **里程碑**: 完成了哪些书籍的学习？
2. **能力提升**: 架构设计能力提升了多少？
3. **调整计划**: 是否需要调整学习顺序？
