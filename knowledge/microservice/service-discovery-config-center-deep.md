# 服务发现与配置中心深度：Consul/Nacos/Etcd 原理与工程实践

> 从服务注册发现到动态配置管理，逐层解析微服务核心基础设施

## 第一部分：服务发现

### 服务发现架构

```
服务发现核心组件：
┌─────────────────────────────────────────────────────────────────────┐
│ 1. 服务提供者                                                          │
│    ├── 注册：启动时向注册中心注册                                      │
│    ├── 续约：定期发送心跳保持健康状态                                   │
│    └── 注销：关闭时向注册中心注销                                     │
│                                                                     │
│ 2. 服务消费者                                                          │
│    ├── 发现：从注册中心获取服务列表                                   │
│    ├── 缓存：本地缓存服务列表                                         │
│    └── 负载：负载均衡选择实例                                       │
│                                                                     │
│ 3. 注册中心                                                            │
│    ├── 存储：服务实例信息                                             │
│    ├── 同步：集群间数据同步                                         │
│    └── 健康检查：实例健康状态检测                                   │
└─────────────────────────────────────────────────────────────────────┘
```

### 主流服务注册中心对比

```
服务注册中心对比：
┌────────────────┬────────────┬────────────┬────────────┐
│     特性       │  Eureka    │  Consul    │  Nacos     │
├────────────────┼────────────┼────────────┼────────────┤
│ 语言           │ Java       │ Go         │ Java       │

│ 配置管理       │ 无         │ 无         │ 有         │
│ 多云支持       │ 无         │ 有         │ 有         │
└────────────────┴────────────┴────────────┴────────────┘
```

## 第四部分：自测题

### Q1: 服务发现的三大核心组件？

**A**: 服务提供者（注册/续约/注销）、服务消费者（发现/缓存/负载）、注册中心（存储/同步/健康检查）。

### Q2: Consul 的三大核心组件？

**A**: Agent（运行在每个节点上的代理）、Server（集群管理节点）、Client（普通节点）。

### Q3: Nacos 相比 Consul 的主要优势？

**A**: 配置管理（内置配置中心）、多云支持（支持多云部署）、AP/CP 切换（可切换一致性模式）。
