## 第二部分：分布式事务

### 分布式事务协议

```
分布式事务协议：
┌─────────────────────────────────────────────────────────────────────┐
│ 1. 两阶段提交（2PC）                                                 │
│    ├── 准备阶段：询问参与者是否可以提交                               │
│    ├── 提交阶段：通知参与者提交或回滚                                 │
│    └── 缺点：阻塞式、单点故障                                         │
│                                                                     │
│ 2. 三阶段提交（3PC）                                                 │
│    ├── 预提交阶段：询问参与者准备情况                                 │
│    ├── 提交阶段：通知参与者正式提交                                   │
│    └── 优点：非阻塞、减少等待时间                                     │
│                                                                     │
│ 3. TCC（Try-Confirm-Cancel）                                         │
│    ├── Try：预留资源                                                   │
│    ├── Confirm：确认提交                                               │
│    └── Cancel：释放预留                                               │
│                                                                     │
│ 4. Saga 模式                                                          │
│    ├── 长事务拆分为短事务序列                                         │
│    ├── 每个事务本地提交                                               │
│    └── 失败时执行补偿事务                                             │
└─────────────────────────────────────────────────────────────────────┘
```

### 分布式事务实现

```
分布式事务实现：
┌─────────────────────────────────────────────────────────────────────┐
│ 1. Seata                                                              │
│    ├── AT 模式：自动补偿                                               │
│    ├── TCC 模式：手动补偿                                               │
│    ├── Saga 模式：长事务                                               │
│    └── XA 模式：强一致性                                               │
│                                                                     │
│ 2. RocketMQ 事务消息                                                  │
│    ├── 半消息：发送但不立即可见                                         │
│    ├── 事务执行：执行本地事务                                         │
│    └── 消息确认：提交或回滚                                           │
│                                                                     │
│ 3. Outbox 模式                                                        │
│    ├── 本地事务：写入业务表+Outbox 表                                 │
│    ├── 变更捕获：CDC 工具监听 Outbox 表                               │
│    └── 消息发送：发送到消息队列                                       │
└─────────────────────────────────────────────────────────────────────┘
```

## 第四部分：自测题

### Q1: API 网关的三大核心功能？

**A**: 路由管理（静态/动态/负载均衡）、安全控制（认证/授权/限流）、监控与日志（指标收集/链路追踪/日志聚合）。

### Q2: 四种分布式事务协议的优缺点？

**A**: 2PC（强一致但阻塞）、3PC（非阻塞但复杂）、TCC（灵活但需手动实现）、Saga（高性能但需补偿逻辑）。

### Q3: API 网关性能优化的三种方法？

**A**: 缓存（响应缓存）、压缩（GZIP/Brotli）、连接池（HTTP/2 多路复用）。
