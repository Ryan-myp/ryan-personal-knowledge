```
分布式事务问题：
```

### TCC 事务模式

```
TCC 三段式事务：
┌─────────────────────────────────────────────────────────────────────┐
│ 1. Try 阶段                                                          │
│    ├── 业务检查                                                       │
│    ├── 预留资源                                                       │
│    └── 锁定数据                                                       │
│                                                                     │
│ 2. Confirm 阶段                                                      │
│    ├── 确认执行                                                       │
│    ├── 提交资源                                                       │
│    └── 释放锁                                                         │
│                                                                     │
│ 3. Cancel 阶段                                                       │
│    ├── 回滚操作                                                       │
│    ├── 释放资源                                                       │
│    └── 清理数据                                                       │
│                                                                     │
│ 特点：                                                               │
│ • 业务侵入性强：需要实现 Try/Confirm/Cancel                          │
│ • 高性能：无全局锁                                                   │
│ • 最终一致性：允许短暂不一致                                         │
│ • 适用场景：高并发、高性能要求的分布式系统                           │
└─────────────────────────────────────────────────────────────────────┘
```

## 第四部分：自测题

### Q1: API 网关的四大核心功能？

**A**: 路由管理、安全控制、可观测性、协议转换。

### Q2: TCC 事务的三个阶段？

**A**: Try（预留资源）、Confirm（提交）、Cancel（回滚）。

### Q3: 2PC/TCC/Saga 的对比？

**A**: 2PC 强一致低性能，TCC 高性能高复杂度，Saga 异步最终一致性。
