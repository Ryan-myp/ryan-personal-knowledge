## 第一部分：服务网格基础

### 什么是服务网格

```
服务网格定义：
┌─────────────────────────────────────────────────────────────────────┐
│ 1. 概念                                                             │
│    ├── 基础设施层：独立于应用代码                                     │
│    ├── 数据平面：Sidecar 代理                                         │
│    └── 控制平面：服务发现/路由/策略                                   │
│                                                                     │
│ 2. 核心价值                                                          │
│    ├── 解耦：业务逻辑与网络通信分离                                   │
│    ├── 透明：无需修改应用代码                                         │
│    └── 统一：跨语言/框架的统一网络层                                  │
│                                                                     │
│ 3. 典型架构                                                          │
│    ├── 应用服务：业务逻辑                                           │
│    ├── Sidecar：代理（Envoy/Istio Proxy）                           │
│    ├── 控制平面：Istiod/Linkerd Controller                          │
│    └── 数据平面：服务间通信                                           │
└─────────────────────────────────────────────────────────────────────┘
```

### 服务网格 vs API 网关

```
服务网格与 API 网关对比：
┌────────────────┬────────────────┬────────────────┐
│     特性       │  服务网格       │  API 网关       │
├────────────────┼────────────────┼────────────────┤
│ 作用范围       │ 服务间通信      │ 外部流量入口    │
│ 部署位置       │ Sidecar/内嵌    │ 独立部署        │
│ 主要功能       │ 负载均衡/熔断   │ 路由/认证/限流  │
│ 协议支持       │ HTTP/gRPC/TCP   │ HTTP/HTTPS     │
│ 配置复杂度     │ 高              │ 中              │
│ 适用场景       │ 微服务内部通信  │ 外部流量管理    │
└────────────────┴────────────────┴────────────────┘
```

## 第二部分：Istio 架构

### Istio 核心组件

```
Istio 核心组件：
┌─────────────────────────────────────────────────────────────────────┐
│ 1. Pilot                                                              │
│    ├── 服务发现：Service Registry                                    │
│    ├── 路由配置：VirtualService/Gateway                              │
│    └── 流量管理：DestinationRule/ServiceEntry                        │
│                                                                     │
│ 2. Mixer                                                              │
│    ├── 策略检查：AuthorizationPolicy                                 │
│    ├── 遥测收集：Telemetry                                           │
│    └── 适配器：Adapter                                               │
│                                                                     │
│ 3. Citadel                                                            │
│    ├── 证书管理：Certificate Management                              │
│    ├── 密钥轮换：Key Rotation                                        │
│    └── 身份认证：Identity Authentication                             │
│                                                                     │
│ 4. Ingress/Egress Gateway                                             │
│    ├── 入站流量：External Access                                     │
│    ├── 出站流量：External Services                                   │
│    └── TLS 终止：TLS Termination                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### Istio 流量管理

```
Istio 流量管理：
┌─────────────────────────────────────────────────────────────────────┐
│ 1. VirtualService                                                     │
│    ├── 路由规则：HTTP/TCP/GRPC                                       │
│    ├── 权重分发：Canary/Blue-Green                                   │
│    └── 故障注入：Delay/Abort                                         │
│                                                                     │
│ 2. DestinationRule                                                    │
│    ├── 负载均衡：RoundRobin/LeastConnection                          │
│    ├── 连接池：TCP/HTTP                                              │
│    └── 出站策略：EgressRule                                         │
│                                                                     │
│ 3. Gateway                                                            │
│    ├── 入站配置：Ingress                                             │
│    ├── 端口绑定：Port Mapping                                        │
│    └── TLS 配置：TLS Mode                                            │
└─────────────────────────────────────────────────────────────────────┘
```

## 第五部分：自测题

### Q1: 服务网格的三大核心价值？

**A**: 解耦（业务逻辑与网络通信分离）、透明（无需修改应用代码）、统一（跨语言/框架的统一网络层）。

### Q2: Istio 的四大核心组件？

**A**: Pilot（服务发现/路由配置）、Mixer（策略检查/遥测收集）、Citadel（证书管理）、Ingress/Egress Gateway（入站/出站流量）。

### Q3: Linkerd 相比 Istio 的主要优势？

**A**: 轻量级（Rust 编写/低内存占用）、易用性（简单安装/最少配置）、安全性（自动 mTLS/Service Account）。
