## 第二部分：人才选拔 — 源码级框架

### 2.1 人才画像 (Person Profile)

```go
// talent_profile.go — 人才画像模型
package management

import (
	"fmt"
	"time"
)

// SkillLevel 技能等级
type SkillLevel int

const (
	SkillJunior  SkillLevel = 1 // 初级
	SkillMid     SkillLevel = 2 // 中级
	SkillSenior  SkillLevel = 3 // 高级
	SkillStaff   SkillLevel = 4 // 专家
	SkillPrincipal SkillLevel = 5 // 资深专家
)

// SkillCategory 技能类别
type SkillCategory string

const (
	SkillTechnical   SkillCategory = "technical"    // 技术能力
	SkillArchitecture SkillCategory = "architecture" // 架构能力
	SkillLeadership  SkillCategory = "leadership"   // 领导力
	SkillBusiness    SkillCategory = "business"     // 业务理解
	SkillCommunication SkillCategory = "communication" // 沟通能力
)

// Skill 技能项
type Skill struct {
	Name    string        `json:"name"`
	Category SkillCategory `json:"category"`
	Level   SkillLevel    `json:"level"`
	Evidence string       `json:"evidence"` // 技能证据
}

// PersonProfile 人才画像
type PersonProfile struct {
	Name        string      `json:"name"`
	Email       string      `json:"email"`
	Role        string      `json:"role"` // 工程师/TL/Lead
	Level       SkillLevel  `json:"level"`
	Skills      []Skill     `json:"skills"`
	Achievements []string   `json:"achievements"`
	Strengths   []string    `json:"strengths"`    // 优势
	Weaknesses  []string    `json:"weaknesses"`   // 待改进
	Goals       []string    `json:"goals"`        // 个人目标
	Availability int        `json:"availability"` // 可投入新项目时间 (小时/周)
	LastReview  time.Time   `json:"last_review"`  // 最近一次绩效评估
	Notes       string      `json:"notes"`        // 备注
}

// TeamMatrix 团队技能矩阵
type TeamMatrix struct {
	Members  []*PersonProfile
	Skills   []string   // 所有技能名称
	Levels   []SkillLevel // 所有等级
}

// BuildTeamMatrix 构建团队技能矩阵
func (m *TeamMatrix) BuildTeamMatrix(team []*PersonProfile) {
	m.Members = team
	
	// 收集所有技能
	allSkills := make(map[string]bool)
	for _, member := range team {
		for _, skill := range member.Skills {
			allSkills[skill.Name] = true
		}
	}
	
	for skill := range allSkills {
		m.Skills = append(m.Skills, skill)
	}
}

// GetSkillDistribution 获取技能分布
func (m *TeamMatrix) GetSkillDistribution() map[string]map[int]int {
	dist := make(map[string]map[int]int)
	
	for _, skillName := range m.Skills {
		dist[skillName] = make(map[int]int)
		for _, member := range m.Members {
			for _, skill := range member.Skills {
				if skill.Name == skillName {
					dist[skillName][int(skill.Level)]++
				}
			}
		}
	}
	
	return dist
}
```

## 第六部分：自测

### Q1：1:1 沟通中，TL 应该如何处理团队成员的压力问题？

<details>
<summary>点击查看参考答案</summary>

**处理步骤**:

1. **倾听为主**: 先让成员表达, 不打断, 不评判
2. **共情**: 表示理解, "我能理解你的感受"
3. **分析根源**: 是工作量大? 技术问题? 人际冲突? 个人问题?
4. **制定行动计划**:
   - 工作量大 → 重新分配任务/增加资源
   - 技术问题 → 提供指导/Mentoring
   - 人际冲突 → 调解/调整协作方式
   - 个人问题 → 提供弹性/资源支持
5. **跟进**: 下周 1:1 检查改善情况

**禁忌**:
- ❌ 忽视或淡化压力 ("别想太多")
- ❌ 立即给解决方案 (先倾听)
- ❌ 泄露隐私 (与其他同事讨论)
- ❌ 承诺做不到的事情

**最佳实践**:
- 建立信任关系 (定期 1:1)
- 关注心理健康 (EAP 资源)
- 营造安全的工作环境
</details>

### Q2：如何判断团队成员是否需要晋升？

<details>
<summary>点击查看参考答案</summary>

**判断标准 (四维度)**:

1. **业绩**: 过去 6 个月持续达到或超过当前 level 的要求
2. **能力**: 已具备下一 level 的核心能力 (通过评估)
3. **影响**: 已经在下一 level 的角色中发挥作用 (Mentoring/系统设计)
4. **文化**: 符合团队价值观, 团队协作良好

**晋升流程**:
1. TL 提名 (基于评估)
2. 技能矩阵对比 (与目标 level 对比)
3. 绩效评估 (过去 2-3 次 1:1 + 季度评估)
4. 跨级评审 (其他 TL/Manager 评审)
5. HR 审批
6. 晋升通知

**关键**: 晋升不是奖励, 是角色转变。要评估候选人是否已经在新角色中表现。
</details>

### Q3：OKR 和 KPI 有什么区别？什么时候用哪个？

<details>
<summary>点击查看参考答案</summary>

**OKR (Objectives & Key Results)**:
- 方向性的, 鼓舞人心的目标
- Key Results 是衡量目标是否达成的关键结果
- 适合创新/探索性工作
- 通常不设奖惩 (避免保守)
- 季度制定, 月度 Review

**KPI (Key Performance Indicators)**:
- 量化的, 可衡量的指标
- 适合成熟/可预测的工作
- 通常与绩效/奖惩挂钩
- 年度或半年度制定

**你的场景建议**:
- TL 角色 → OKR 为主 (技术路线图/团队建设)
- 工程师个体 → OKR 为主 (个人成长/项目交付)
- 团队指标 → OKR + KPI 混合 (团队交付 + 性能指标)

**最佳实践**: 用 OKR 设定方向, 用 KPI 跟踪日常运营。
</details>
