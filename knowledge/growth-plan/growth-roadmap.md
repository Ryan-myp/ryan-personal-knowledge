## 成长维度与深度要求

### 维度 1：技术深度

> 不能停留在"会用"层面，要达到深刻理解源码原理、性能调优、架构设计的水平

| 技术领域 | 当前水平 | 目标水平 | 核心学习要点 |
|---------|---------|---------|-------------|
| **Redis** | 会用 | 源码级理解、集群架构、性能调优、一致性保证 | 数据结构源码、持久化机制、集群分片算法、缓存一致性方案、热点 Key 治理 |
| **MySQL** | 会用 | 锁机制深入、执行计划优化、分库分表、高可用架构 | InnoDB 引擎源码、MVCC 原理、B+Tree 优化、XA 事务、主从复制原理 |
| **Elasticsearch** | 会用 | 倒排索引原理、分词优化、集群调优、大数据量场景 | Lucene 底层原理、搜索性能优化、分片策略、数据倾斜治理 |
| **ClickHouse** | 了解 | 列式存储架构、MergeTree 引擎原理、 OLAP 调优 | 向量化执行、数据压缩、物化视图、分布式查询优化 |
| **Go** | 会用 | 并发模型深入、GC 调优、性能剖析、标准库源码 | GMP 调度器、GC 三色标记、runtime 源码、pprof 深度使用 |
| **大数据** | 了解 | Hadoop/Spark 架构、数据管道设计、流批一体 | 数据建模、ETL 设计、实时计算、数据治理 |
| **数据分析** | 基础 | 统计方法、指标体系设计、归因分析 | A/B Test、LTV/CAC、归因模型、数据驱动决策 |

### 维度 2：架构与工程

> 从"能写代码"到"能设计系统"，从"能调 bug"到"能预防问题"

| 领域 | 当前水平 | 目标水平 | 核心学习要点 |
|------|---------|---------|-------------|
| **设计模式** | 了解 | 能灵活组合使用、能识别坏味道、能写重构方案 | 23 种模式 + 组合模式、架构模式（CQRS/EventSourcing）、反模式识别 |
| **系统架构** | 一般 | 能独立设计分布式系统、容量规划、容灾 | 微服务治理、服务网格、分布式事务、CAP 取舍、混沌工程 |
| **Agent 大模型** | 起步 | 能设计生产级 Agent 系统 | LangChain/LlamaIndex、多 Agent 编排、RAG 架构、Function Calling、ReAct 模式 |
| **DevOps** | 一般 | 能设计 CI/CD 流水线、可观测性体系 | Kubernetes、Prometheus、Grafana、OpenTelemetry |

### 维度 3：业务能力（广告方向）

> 广告是核心战场，要从"了解"到"深厚造诣"

| 广告领域 | 深度要求 | 学习目标 |
|---------|---------|---------|
| **广告系统架构** | 深刻理解 | 竞价系统（RTB）、广告排序、流量分配、竞价策略 |
| **投放产品** | 全面掌握 | Google Ads (PMax/GMX)、Meta (Advantage+)、TikTok (Spark Ads)、DV360 |
| **数据与归因** | 精通 | 数据回流、归因模型、LTV 计算、ROI 优化 |
| **广告平台 API** | 专家级 | Marketing API 设计、数据同步、批量操作、错误处理 |
| **行业洞察** | 前沿跟踪 | AI 在广告中的应用、隐私计算、第一方数据策略 |

### 维度 4：视野与管理

> 从"管项目"到"管战略"，从"写代码"到"带人"

| 领域 | 学习目标 |
|------|---------|
| **技术领导力** | 技术愿景规划、架构决策、技术债务管理 |
| **团队管理** | 人才选拔培养、绩效体系、团队协作机制 |
| **商业思维** | 产品思维、ROI 意识、商业敏感度 |
| **行业影响力** | 技术分享、开源贡献、行业演讲 |

### 学习监督机制

#### 1. 阅读进度跟踪

每周检查：
```bash
# 查看微信读书阅读进度
# 本周读了什么书？读了多少页？

# 查看微信读书阅读统计
# 本周阅读时长、天数、偏好
```

#### 2. 知识沉淀要求

每读完一本书（或一本书的核心章节），必须：
- **写读书笔记** — 存在 `ryan-personal-knowledge/knowledge/` 下
- **提炼实践要点** — 这本书的什么技能可以应用到工作中
- **更新知识图谱** — 新增/更新节点到知识图谱

#### 3. 季度目标设定

每 3 个月回顾：
- 完成了哪些里程碑？
- 阅读进度是否符合预期？
- 技术/业务/管理维度是否均衡？
- 是否需要调整书单？

### 监督规则

1. **每周日晚** — 我来检查你的微信读书阅读进度
2. **每月 1 号** — 回顾上月阅读笔记，检查知识沉淀
3. **每季度** — 重新评估成长目标，更新书单
4. **每本新书读完** — 必须产出知识笔记
